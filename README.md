# V8 wasm relaxed-SIMD FMA regression

Minimal, self-contained repro for a large performance regression in V8's
WebAssembly compilation of relaxed-SIMD FMA-heavy code.

## TL;DR

Same ~90-line wasm function, same machine, same input:

| engine | tier | code size | throughput |
|---|---|---|---|
| V8 12.9.202 (Node 23.11) | TurboFan | 892 bytes | **11.1 GFMA/s** (0.09 ns/FMA) |
| V8 13.6.233 (Node 24.15) | Turboshaft | 2112 bytes (**2.37×**) | **2.2 GFMA/s** (0.45 ns/FMA) |
| V8 14.9.181 (latest canary) | Turboshaft | — | 2.2 GFMA/s (not fixed) |

→ **~5× slowdown**, introduced when Turboshaft became the default Wasm
optimizing pipeline (V8 13.2.51, [CL 5953274][cl]). Not yet fixed in latest.

[cl]: https://chromium-review.googlesource.com/c/v8/v8/+/5953274

## What pattern triggers it

The benchmarked function mirrors a 51x5 Montgomery-multiply inner loop:

- 16 v128 locals live at once (5 × `Y`, 5 × `LH` scratch, 5 × `Z` acc, 1 × `xi`)
- Per loop iteration, 5 passes of:
  - 5 × `f64x2.relaxed_madd(xi, Y[j], C103)` → write `LH[j]` (hi)
  - 5 × `Z[j] = i64x2.add(Z[j], LH[j])`
  - 5 × `LH[j] = f64x2.sub(C2, LH[j])` (reuse scratch)
  - 5 × `f64x2.relaxed_madd(xi, Y[j], LH[j])` → write `LH[j]` (lo)
  - 5 × `Z[j] = i64x2.add(Z[j], LH[j])`
- 50 FMAs per outer iteration, 100M FMAs total.

**A plain long chain of `f64x2.relaxed_madd` does NOT reproduce** (Node 24 is
actually slightly *faster*). The regression requires the full interleaved
shape: mixed SIMD op kinds + register pressure + scratch-local reuse.

### Mechanism (from the disassembly diff)

Instruction counts in the optimized tier on the same function:

| op | V8 12.9 TurboFan | V8 13.6 Turboshaft |
|---|---|---|
| `vfmadd*pd` (total) | 10 | 50 |
| `vsubpd` | 5 | 25 |
| `vmovups` (spill/reload) | 37 | 37 |

Spill counts are **identical**. The 5× blow-up is not a regalloc regression;
Turboshaft is **fully unrolling the inner `p` loop** (P=5 passes), emitting
all 50 FMAs inline where TurboFan keeps the loop rolled with 10 FMAs in a
tight body. The unrolled version has long `Z[j]` read-modify-write chains
across the 5 unrolled copies that serialize the FMA pipe; the rolled version
hides that latency in the loop backedge.

## How to reproduce

Requires Node 23+ (for `--experimental-strip-types`) and `n` (or similar)
to switch between Node versions.

```bash
npm install

# 1) Node-side benchmark (also writes build.wasm that d8 loads later)
n 23 && npm run bench   # → ~11 GFMA/s
n 24 && npm run bench   # → ~2.2 GFMA/s

# 2) Install d8 via jsvu (takes a few min, downloads V8 binaries)
npm run jsvu-install

# 3) Standalone d8 benchmarks — same wasm, no Node at all
npm run bench-d8-old      # V8 12.9.202  → ~11 GFMA/s
npm run bench-d8-new      # V8 13.6.233  → ~2.2 GFMA/s
npm run bench-d8-latest   # latest canary → still ~2.2 GFMA/s

# 4) Dump x86-64 disassembly for the optimizing tier on both versions
npm run disasm-both
diff dump/v8-12.9-turbofan.asm dump/v8-13.6-turboshaft.asm | less
```

Diffing the two `.asm` files shows exactly what Turboshaft does differently
(expected: more `vmovups xmm, [rbp-…]` spills, worse instruction scheduling).

## Files

- `repro.ts` — uses [wasmati][wasmati] to build the module + runs the bench in
  Node; also writes `build.wasm` to disk for the d8 workflow.
- `harness.js` — plain JS harness that loads `build.wasm` and runs the bench.
  Works in d8 (uses `read()` / `print()` built-ins). 
- `build.wasm` — generated by `npm run bench`, checked in? no (gitignored).

[wasmati]: https://github.com/mitschabaude/wasmati

## Related

- Upstream Node issue: https://github.com/nodejs/node/issues/61695 — a different
  trigger (plain `v128` base64 decoder, ~57% regression), but same root cause.
  Reporter bisected to V8 13.2.51 and confirmed `--no-turboshaft-wasm` restores
  perf. That flag has since been removed from V8.
- Turboshaft tracking bug: https://issues.chromium.org/issues/42204049

## Environment

Numbers in this README: x86-64 Linux (kernel 6.8), AMD Ryzen-class CPU with
AVX2+FMA3. Your numbers will differ in absolute value; the 5× delta between
V8 12.9 and V8 13.6 on the same machine is the signal.
